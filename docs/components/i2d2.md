# i2d2 — Component Design
**Status:** INPROGRESS | **Folder:** `i2d2/`

---

## Role

i2d2 (Intelligent Infrastructure Design Decision) is the reasoning engine and pipeline orchestrator. It owns the full transformation from NLP to Kanvas — and coordinates every component in between. It is the only component that writes artifacts to katalog.

---

## Responsibilities

- **Kit extraction** — calls konnekt, validates LLM response against Kit schema (Pydantic)
- **kick resolution** — reads klue registry directly as data, infers applicable klue IDs via its own LLM call. Checks katalog for existing matching kick before re-resolving.
- **kg1 gate** — calls konform(kit, kick_id); writes kit + verdict to katalog on pass
- **skout + skan coordination** — optionally summons skout and skan for design context pre-kraph. Not pipeline stages — summoned as needed.
- **Kraph generation** — two paths:
  - (a) skout findings sufficient → retrofit existing kraph/kanvas (sufficiency logic TBD)
  - (b) skout findings insufficient → LLM call: (kit + kick + skout findings + skan findings) → kraph
  - DAG validation + enum validation run as Pydantic model validators at construction time
  - Retry loop max 2 on validation failure — beyond that → design_conflicts[]
  - Mermaid DSL generated deterministically from kraph struct — no LLM call. Written to katalog atomically with kraph.
  - Writes kraph to katalog
- **Kanvas assembly** — resolves konfig values and assembles full infrastructure manifest directly. No separate knit/kwery components.
- **Kanvas persistence** — writes kanvas to katalog pre-kg2
- **kg2 gate** — calls konform(kanvas, kick_id); appends verdict to katalog
- **koder invocation** — passes kanvas to koder → HCL; koder writes HCL to katalog
- **Complexity routing** — detects `complexity_flags` in Kit, triggers Opus when above threshold (scoring logic deferred)

## Kit Schema — Signal Types

```
KitSignal
  value: str
  source_span: str | None
  inferred: bool = False

SizeSignal
  metric: str          # "rps", "storage", "concurrent_users", etc.
  value: str           # as-is, no normalization
  source_span: str | None
  inferred: bool = False

NfrSignal
  metric: str          # "latency", "availability", "throughput", etc.
  value: str           # as-is — "fast", "99.99%", "100ms"
  source_span: str | None
  inferred: bool = False

TemporalSignal
  pattern_type: str    # "peak_window", "schedule", "burst"
  value: str
  source_span: str | None
  inferred: bool = False

ExclusionSignal
  target: str          # "kubernetes", "lambda", etc.
  reason: str | None   # only if stated or clearly implied
  source_span: str | None
  inferred: bool = False

IntentType (enum)
```

## Design Decisions

- `kit_id` auto-generated UUID — LLM never provides it
- MVP handles `PROVISION` intent only — `MODIFY` defined but not yet handled at runtime
- Kit never normalizes signal values — resolved by i2d2 + skope during Kanvas assembly
- cost_posture is a design signal, not a policy signal — cost enforcement is Gate 2 only
- DAG validation is i2d2's responsibility — Pydantic model validator on kraph, not konform
- i2d2 is the only writer to katalog — konform never writes to persistent stores
- kraken flag travels kit → kick → kraph → kanvas — konform gates run normally regardless
- kick resolution is i2d2-internal — no separate klue component
