# i2d2 — Component Design
**Status:** INPROGRESS | **Folder:** `i2d2/` | **Phase:** 1 MVP

---

## Role

i2d2 (Intelligent Infrastructure Design Decision) is the reasoning engine and pipeline orchestrator. It owns the full transformation from NLP to Kanvas — and coordinates every component in between. It is the only component that writes artifacts to katalog.

---

## Responsibilities

- **Kit extraction** — calls konnekt, validates LLM response against Kit schema (Pydantic)
- **klue invocation** — passes (kit, skope) to klue to infer kick
- **kg1 gate** — calls konform(kit, kick_id); writes kit + verdict to katalog on pass
- **skout + skan coordination** — passes (kit_id, kick_id) to skout and skan for design aid findings
- **Krux generation** — two paths:
  - (a) skout findings sufficient → retrofit existing krux/kanvas (sufficiency logic TBD — ADR-037)
  - (b) skout findings insufficient → LLM call: (kit + kick + skout findings + skan findings) → krux
  - DAG validation + enum validation run as Pydantic model validators at construction time
  - Retry loop max 2 on validation failure — beyond that → design_conflicts[]
  - Writes krux to katalog
- **knit invocation** — passes krux to knit; knit calls kwery internally → produces kanvas
- **Kanvas persistence** — writes kanvas to katalog pre-kg2 (ADR-035)
- **kg2 gate** — calls konform(kanvas, kick_id); appends verdict to katalog
- **koder invocation** — passes kanvas to koder → HCL; koder writes HCL to katalog
- **Complexity routing** — detects `complexity_flags` in Kit, triggers Opus when above threshold (scoring logic parked)

## Kit Schema — Signal Types

```
KitSignal
  value: str
  source_span: str | None
  inferred: bool = False

SizeSignal
  metric: str          # "rps", "storage", "concurrent_users", etc.
  value: str           # as-is, no normalization
  source_span: str | None
  inferred: bool = False

NfrSignal
  metric: str          # "latency", "availability", "throughput", etc.
  value: str           # as-is — "fast", "99.99%", "100ms"
  source_span: str | None
  inferred: bool = False

TemporalSignal
  pattern_type: str    # "peak_window", "schedule", "burst"
  value: str
  source_span: str | None
  inferred: bool = False

ExclusionSignal
  target: str          # "kubernetes", "lambda", etc.
  reason: str | None   # only if stated or clearly implied
  source_span: str | None
  inferred: bool = False

IntentType (enum)
  PROVISION | MODIFY | DESTROY | QUERY
  MVP runtime handles PROVISION only

Kit
  request_id: UUID                          # auto-generated
  raw_input: str                            # original NLP, untouched
  intent: IntentType
  kraken: bool = False                      # pipeline flag — ADR-038
  explicit_infra: list[KitSignal]           = []
  app_type: list[KitSignal]                 = []
  qualifiers: list[KitSignal]               = []
  size_and_scale: list[SizeSignal]          = []
  nfr_targets: list[NfrSignal]             = []
  temporal_pattern: list[TemporalSignal]    = []
  lifecycle: list[KitSignal]               = []
  data_characteristics: list[KitSignal]    = []
  access_pattern: list[KitSignal]          = []
  security_posture: list[KitSignal]        = []
  cost_posture: list[KitSignal]            = []
  explicit_constraint: list[KitSignal]     = []
  exclusions: list[ExclusionSignal]        = []
  complexity_flags: list[str]              = []
```

## Design Decisions

- `inferred_implicit` retired as a category — inference is `inferred: bool` field on every signal (ADR-014)
- All list fields default to `[]` — no None checks downstream
- `request_id` auto-generated UUID — LLM never provides it
- MVP handles `PROVISION` intent only — other IntentTypes defined but rejected at runtime
- Kit never normalizes signal values — resolved by i2d2 + skope during Kanvas design
- cost_posture is a design signal, not a policy signal — cost enforcement is Gate 2 only (ADR-013)
- DAG validation is i2d2's responsibility — Pydantic model validator on krux, not konform (ADR-024)
- i2d2 is the only writer to katalog — konform never writes to persistent stores (ADR-036)
- kraken flag travels kit → kick → krux → kanvas — konform gates run normally regardless (ADR-038)
