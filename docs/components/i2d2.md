# i2d2 ‚Äî Component Design
**Status:** INPROGRESS | **Folder:** `i2d2/`

---

## Role

i2d2 (Intelligent Infrastructure Design Decision) is the reasoning engine and pipeline orchestrator. It owns the full transformation from NLP to Kanvas ‚Äî and coordinates every component in between. It is the only component that writes artifacts to katalog.

---

## Responsibilities

- **Kit extraction** ‚Äî calls konnekt, validates LLM response against Kit schema (Pydantic)
- **kick resolution** ‚Äî reads klue registry directly as data, infers applicable klue IDs via its own LLM call. Checks katalog for existing matching kick before re-resolving.
- **kg1 gate** ‚Äî calls konform(kit, kick_id); writes kit + verdict to katalog on pass
- **skout + skan coordination** ‚Äî optionally summons skout and skan for design context pre-kraph. Not pipeline stages ‚Äî summoned as needed.
- **Kraph generation** ‚Äî two paths:
  - (a) skout findings sufficient ‚Üí retrofit existing kraph/kanvas (sufficiency logic TBD)
  - (b) skout findings insufficient ‚Üí LLM call: (kit + kick + skout findings + skan findings) ‚Üí kraph
  - DAG validation + enum validation run as Pydantic model validators at construction time
  - Retry loop max 2 on validation failure ‚Äî beyond that ‚Üí design_conflicts[]
  - Mermaid DSL generated deterministically from kraph struct ‚Äî no LLM call. Written to katalog atomically with kraph.
  - Writes kraph to katalog
- **Kanvas assembly** ‚Äî resolves konfig values and assembles full infrastructure manifest directly. No separate knit/kwery components.
- **Kanvas persistence** ‚Äî writes kanvas to katalog pre-kg2
- **kg2 gate** ‚Äî calls konform(kanvas, kick_id); appends verdict to katalog
- **koder invocation** ‚Äî passes kanvas to koder ‚Üí HCL; koder writes HCL to katalog
- **Complexity routing** ‚Äî detects `complexity_flags` in Kit, triggers Opus when above threshold (scoring logic deferred)

## Design Decisions

- `kit_id` auto-generated UUID ‚Äî LLM never provides it
- MVP handles `PROVISION` intent only ‚Äî `MODIFY` defined but not yet handled at runtime
- Kit never normalizes signal values ‚Äî resolved by i2d2 + skope during Kanvas assembly
- cost_posture is a design signal, not a policy signal ‚Äî cost enforcement is Gate 2 only
- DAG validation is i2d2's responsibility ‚Äî Pydantic model validator on kraph, not konform
- i2d2 is the only writer to katalog ‚Äî konform never writes to persistent stores
- kraken flag travels kit ‚Üí kick ‚Üí kraph ‚Üí kanvas ‚Äî konform gates run normally regardless
- kick resolution is i2d2-internal ‚Äî no separate klue component
