# ARC

## Overview
ARC is the Architecture Catalog of kre8.

It serves both as:
1. a record and retention layer for architecture outputs
2. a browseable selection layer where users can choose existing designs or starting points

## Purpose
ARC allows kre8 to move beyond one-off generation and into reuse, curation, and accelerated delivery.

## What ARC Enables
Users should be able to:
- select an existing Design Plan as-is
- select an existing Design Plan and request modifications
- browse reusable architecture options
- start from proven baseline patterns rather than from scratch

## Product Role
ARC can participate:
- before generation, through lookup and suggestion
- after generation, through storage and future reuse

## Relationship to Other Components
- surfaced through Kiosk
- influenced by Realm scoping
- used by Think when a prior Design Plan becomes the baseline for change intent
